Every value in Noise is a probability distribution, and the only way to get a number out is to
**run** it. That makes for a frustrating static blog post — so posts here aren't static. They're
[MDX](https://mdxjs.com/): Markdown that can drop a live, interactive component into the middle of
a paragraph.

This post is also the template. Copy it, rename it, write.

## Just write Markdown

A post is one file in `src/content/blog/`. Plain `.md` works when all you want is prose, headings,
links, and code fences. A fenced `noise` block is highlighted with the same grammar as the editor:

```noise
flips ~ rand::bernoulli(0.5)     # a fair coin: 1 = heads, 0 = tails
heads = sum(~[100] flips)        # draw 100, count the heads
E(heads)                         # ≈ 50
```

Rename the extension to `.mdx` and you can import components and render them inline. A few are
**already available in every post** — no import needed.

## Drop in a code panel

`<CodePanel>` is the same listing component the home page uses, with an "open in playground" link
wired up for free:

<CodePanel
  filename="coin.noise"
  code={`p = 0.5
heads = sum(~[1000] rand::bernoulli(p))
E(heads) / 1000   # the long-run proportion → p`}
/>

## Embed one of the home-page demos

The scroll-driven demos from the landing page are just components. Write the tag and you get the
whole thing — sticky canvas, narrated steps, and a real engine run at the end:

<ConceptDemo />

`<CircleDemo />`, `<BirthdayDemo />`, `<BellDemo />`, `<AmFmDemo />` and `<PrisonersDemo />` work

## Build your own scrolly

For a custom story, compose `<Scrolly>` (a sticky stage + narrated steps) with `<ScrollyStep>`.
As each step scrolls into view, the `<Scrolly>` root fires a `scrolly:step` event — wire it to a
canvas, an SVG, or a live WASM run. The demo below redraws the **running proportion of heads** as
the sample size grows, so you can watch the [law of large
numbers](https://en.wikipedia.org/wiki/Law_of_large_numbers) pull a jagged line flat at ½:

<LargeNumbersDemo />

The structure — the `<Scrolly>` and its `<ScrollyStep>`s — is plain MDX. One catch: the visual's
JavaScript lives in a small `.astro` component
([`LargeNumbersDemo.astro`](https://github.com/manucorporat/noise-lang/blob/master/www/src/components/LargeNumbersDemo.astro)),
not inline in this file — MDX reads `{` and `<` inside a `<script>` as JSX, so real code has to be
bundled by a component. The skeleton an author writes is just:

```jsx
<Scrolly caption="…">
  <div slot="stage">{/* a <canvas> the script draws into */}</div>
  <ScrollyStep n={0} title="…">First beat of the story.</ScrollyStep>
  <ScrollyStep n={1} title="…">Second beat.</ScrollyStep>
</Scrolly>
```

That's the whole toolkit: Markdown for words, `<CodePanel>` for listings, the ready-made demos for
quick embeds, and `<Scrolly>` + a tiny component when you want something new.
